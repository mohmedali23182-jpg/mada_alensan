# ترقية قاعدة البيانات لمنصة مدى الناس

تم استبدال نموذج قاعدة البيانات البدائي بنموذج CMS إنتاجي متقدم داخل `prisma/schema.prisma` مع الحفاظ على توافق الكود الحالي.

## ماذا تغير؟

- إضافة نموذج منظمة `Organization` لعزل إعدادات المنصة مستقبلًا.
- تطوير `Post` ليصبح كيان تحرير كامل لا مجرد جدول مقال.
- إضافة `PostSeo` كسجل SEO مستقل مع إبقاء حقول SEO القديمة للتوافق.
- إضافة `PostRevision` لحفظ نسخ المقالات وتاريخ التعديلات.
- إضافة `PostWorkflowEvent` لتتبع دورة التحرير: إنشاء، مراجعة، اعتماد، نشر، أرشفة.
- إضافة `PostBlock` لدعم المقالات التفاعلية مستقبلًا: فقرة، صورة، اقتباس، فيديو، جدول، استطلاع.
- إضافة `PostStats`, `PostView`, `Comment`, `PostReaction`, `Bookmark` للإحصائيات والتفاعل.
- إضافة `Page`, `Menu`, `MenuItem`, `Redirect`, `NewsletterSubscriber` لإدارة الصفحات والقوائم والتحويلات والنشرات.
- تحسين `Media` وإضافة `PostMedia` لاستخدام الصور كغلاف، مصغرة، داخل المقال، معرض، مرفق.
- تحسين التصنيفات لتدعم التصنيفات الأب/الفرعية.
- تحسين seed ليبني منصة نظيفة: منظمة، أدمن، تصنيفات، مناطق، صفحات، قائمة رئيسية، إعدادات SEO وسياسة تحرير.

## أوامر التطبيق على قاعدة Supabase

عند استخدام قاعدة موجودة وتريد استبدال بنيتها بالبنية الجديدة:

```bash
npm install
npx prisma generate
npx prisma db push --accept-data-loss
npm run seed
```

إذا كانت القاعدة تحتوي جداول قديمة لا تريدها، استخدم ملف:

```txt
supabase/reset-project-tables.sql
```

ثم شغّل:

```bash
npx prisma db push --accept-data-loss
npm run seed
```

## ملاحظة مهمة

`prisma db push --accept-data-loss` قد يحذف أو يعدل أعمدة وجداول غير متوافقة. استخدم نسخة احتياطية قبل تشغيله على قاعدة مهمة.

## التحقق المحلي

تم تنفيذ:

```bash
npm run lint -- --max-warnings=0
npx tsc --noEmit
```

تعذر تنفيذ `prisma validate` و `next build` كاملين داخل هذه البيئة لأن تحميل Prisma engines وخدمات Google Fonts الخارجية غير متاح من الحاوية، لذلك أزيل الاعتماد على `next/font/google` حتى لا يفشل البناء بسبب الخطوط. في بيئة Vercel/Node مع اتصال إنترنت وقاعدة صحيحة، استخدم الأوامر أعلاه.
