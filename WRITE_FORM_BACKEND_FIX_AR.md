# إصلاح إرسال المقال من صفحة /write

تم تنفيذ إصلاحات محافظة لا تغيّر التصميم ولا Telegram webhook ولا تسجيل الدخول ولا بنية قاعدة البيانات.

## ما تم إصلاحه

- جعل رسائل التحقق في نموذج إرسال المقال عربية وواضحة بدل رسائل Zod الإنجليزية مثل:
  - `String must contain at least 20 character(s)`
- جعل حقل `authorBio` / نبذة الكاتب اختياريًا فعلًا.
  - إذا تُرك فارغًا لا يعطي خطأ.
  - إذا كُتب يجب أن يحتوي 5 أحرف على الأقل.
- ضبط قواعد نموذج إرسال المقال:
  - الاسم الكامل مطلوب، حرفان على الأقل.
  - البريد الإلكتروني مطلوب وبصيغة صحيحة.
  - رقم واتساب اختياري.
  - نبذة الكاتب اختيارية.
  - عنوان المقال مطلوب، 5 أحرف على الأقل.
  - نص المقال مطلوب، 20 حرفًا على الأقل.
- تحديث API إرسال المقال:
  - `POST /api/submissions/article`
  - `POST /api/v1/submissions`
- جعل API يرجع JSON منظمًا عند خطأ التحقق:

```json
{
  "ok": false,
  "message": "يرجى مراجعة الحقول المطلوبة.",
  "errors": {
    "body": "نص المقال يجب أن يحتوي على 20 حرفًا على الأقل"
  }
}
```

- تحسين الواجهة لتعرض خطأ الحقل بجانب الحقل نفسه، مع صندوق خطأ واضح أعلى النموذج.
- منع إعادة الإرسال أثناء حالة التحميل.
- بعد النجاح تظهر رسالة نجاح واضحة ويتم تفريغ النموذج.
- إبقاء الحفظ في جدول `Submission` كما هو.
- إبقاء إشعار البريد/تليجرام للأدمن كما هو، مع عدم إفشال الحفظ إذا فشل الإشعار.
- إزالة العبارات التي تُظهر أن الكاتب جاء من بوت تليجرام، واستبدال ملاحظات الإنشاء بعبارة محايدة: `أداة الإدارة`.

## طريقة الاختبار

1. افتح:
   `/write`
2. أرسل مقالًا بنص أقل من 20 حرفًا.
   - يجب أن تظهر رسالة عربية واضحة بجانب حقل نص المقال.
3. املأ البيانات بشكل صحيح وأرسل.
   - يجب أن تظهر رسالة نجاح.
4. افتح Supabase أو لوحة الأدمن وتأكد أن الإرسال ظهر في جدول/قسم الوارد `Submission`.

## ملاحظات مهمة

لا توجد حاجة لتشغيل SQL جديد لأن الإصلاح لا يضيف جداول ولا يغيّر schema.prisma.

بعد رفع النسخة شغّل:

```bash
npm install
npx prisma generate
npm run lint -- --max-warnings=0
npx tsc --noEmit
npm run build
```
