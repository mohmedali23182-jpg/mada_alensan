# تقرير تحسين منصة مدى الإنسان

## ما تم تنفيذه

- إزالة الاعتماد على أي صور وهمية خارجية للكتّاب أو الأغلفة.
- استخدام قالب غلاف محلي عند غياب صورة المقال بدل اختيار صورة من الإنترنت.
- استخدام أيقونة مستخدم/حرف الاسم عند غياب صورة الكاتب بدل `pravatar` أو أي مصدر خارجي.
- ضبط اسم المنصة إلى: **مدى الإنسان** في النصوص والهوية.
- اعتماد الوصف: **إنسانية - اجتماعية - ثقافية - علمية - متنوعة**.
- جعل رفع الصور يستخدم bucket افتراضي باسم `media`.
- إضافة ملف إنشاء bucket:
  - `supabase/02-create-storage-bucket.sql`
- تحسين نموذج إرسال المقال `/write`:
  - رسائل تحقق عربية واضحة.
  - قراءة JSON آمنة.
  - نبذة الكاتب اختيارية فعلًا.
  - حفظ المقال في `Submission`.
  - الرد دائمًا بصيغة JSON.
- تحسين محرر الأدمن:
  - توضيح مقاس الغلاف المطلوب.
  - فحص أبعاد غلاف المقال قبل الرفع.
  - منع رفع أغلفة صغيرة أو غير متناسبة.
- تحسين بوت تليجرام:
  - عدم إنشاء كاتب بعبارة "بوت تليجرام".
  - إضافة رابط المقال الرسمي ورابط التعديل في رسالة النشر.
  - منع إعادة إنشاء slug مختلف في canonical عند النشر من البوت.
  - استخدام `TELEGRAM_ARTICLES_CHANNEL_ID` ثم `TELEGRAM_CHANNEL_ID` ثم `TELEGRAM_CACHE_CHANNEL_ID` حسب المتوفر.
- إضافة مسارات تحويل آمنة لتقليل 404 في الإدارة:
  - `/admin/posts` → `/admin/articles`
  - `/admin/editor` → `/admin/articles`

## مقاس صورة الغلاف المعتمد

- الأفضل: `1600 × 900 px`
- الحد الأدنى المقبول: `1200 × 675 px`
- النسبة المطلوبة: `16:9`
- الصيغ المناسبة: `WebP`, `JPG`, `PNG`
- الحجم المفضل: أقل من `500KB` عند الإمكان، والحد الأعلى في النظام `50MB`.

## ملاحظات Supabase

إذا ظهرت رسالة `Bucket not found` شغّل الملف التالي في Supabase SQL Editor:

```txt
supabase/02-create-storage-bucket.sql
```

وتأكد من وجود المتغيرات:

```env
SUPABASE_STORAGE_BUCKET=media
NEXT_PUBLIC_SUPABASE_STORAGE_BUCKET=media
```

## الفحوصات

تم تشغيل:

```bash
npm run lint -- --max-warnings=0
```

ونجح بدون تحذيرات أو أخطاء.

تعذر تشغيل `prisma generate` و `tsc` كاملًا داخل هذه البيئة لأن Prisma حاول تحميل engines من `binaries.prisma.sh` وفشل الاتصال. على Vercel سيعمل `prisma generate` أثناء البناء إذا كانت الشبكة متاحة.
